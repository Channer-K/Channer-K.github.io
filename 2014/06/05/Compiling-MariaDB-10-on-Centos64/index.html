CentOS 6.4下编译安装MariaDB10

为系统做好编译准备

在安装之前需要到MariaDB的官网下载所需的源码包mariadb-10.0.11.tar.gz

安装依赖

官网给出以下依赖程序，你可以在安装前把下列程序都安装了，也可以等编译出错的时候根据提示来安装相应缺少的依赖，当然如果这样的话你需要多次重复编译过程，也会浪费不少的时间。所以，我们先确保下列程序已经全部安装再开始编译。

  • gunzip
  • GNU tar
  • gcc 2.95.2 or later
  • g++
  • GNU make 3.75 or later
  • libtool 1.5.24 or later
  • bison (2.0 for MariaDB 5.5)
  • libncurses
  • zlib-dev
  • GNU automake
  • GNU autoconf
  • cmake

开始编译安装

默认完全安装

MariaDB官方考虑的很周全，已经在源码包中提供了一些默认的编译脚本，如果没有特殊编译需求的话，简单的执行相应编译脚本，即可告别手动编译过程中一大堆配置项。

解压源码包并进入到源代码目录，在BUILD目录下面可以看到非常多的编译脚本文件，比较常用的有以下几个：

# 为64位intel/amd编译（带所有选项）
BUILD/compile-pentium64-max

# 为32位pentum编译（带所有选项）
BUILD/compile-pentium-max

# 为64位intel/amd编译（带调试选项）
BUILD/compile-pentium64-debug-max

# 为64位intel/amd编译测试（带valgrind）   
BUILD/compile-pentium64-valgrind-max

通过这种方式安装的MariaDB，默认的安装目录在/usr/local/mysql下

执行上述脚本中的一个，经过一段漫长的编译过程之后，终于完成了编译，现在可以开始安装了。

sudo make install

安装完成之后，在启动MariaDB之前应确保：

  1. /usr/local/mysql目录为mysql用户所有
  2. 已完成系统初始化

下面我们就来解决以上两个问题

groupadd mysql
useradd -d /usr/local/mysql -s /sbin/false -g mysql mysql
chown -R mysql.mysql /usr/local/mysql
chown -R mysql.root /var/run/mysqld
cd /usr/local/mysql
scripts/mysql_install_db --user=mysql
cp support-files/mysql.server /etc/init.d/mysqld
chkconfig --add mysqld
chkconfig mysqld on
chmod +x /etc/init.d/mysqld
service mysqld start

自定义编译安装

如果不希望完全安装，或者不想安装到/usr/local/下，就需要手动指定编译项了。

下面给出一个参考样例：

cmake \
# 安装根目录
-DCMAKE_INSTALL_PREFIX=/app/mariadb \
# 数据存储目录
-DMYSQL_DATADIR=/app/mariadb/data/ \
# * ARCHIVE 引擎支持
-DWITH_ARCHIVE_STORAGE_ENGINE=1 \
# * ARIA 引擎支持
-DWITH_ARIA_STORAGE_ENGINE=1 \
# * BLACKHOLE 引擎支持
-DWITH_BLACKHOLE_STORAGE_ENGINE=1 \
# * FEDERATEDX 引擎支持
-DWITH_FEDERATEDX_STORAGE_ENGINE=1 \
# * PARTITION 引擎支持
-DWITH_PARTITION_STORAGE_ENGINE=1 \
# * PERFSCHEMA 引擎支持
-DWITH_PERFSCHEMA_STORAGE_ENGINE=1 \
# * SPHINX 引擎支持
-DWITH_SPHINX_STORAGE_ENGINE=1 \
# * XTRADB 支持
-DWITH_XTRADB_STORAGE_ENGINE=1 \
# * innoDB 引擎支持
-DWITH_INNOBASE_STORAGE_ENGINE=1 \
# * Myisam 引擎支持
-DWITH_MYISAM_STORAGE_ENGINE=1 \
# readline库
-DWITH_READLINE=1 \
# 启用加载本地数据
-DENABLED_LOCALS=1 \
# 扩展支持编码 ( all | utf8,gbk,gb2312 | none )
-DWITH_EXTRA_CHARSETS=all

更多编译参数可以通过cmake . -L来查看。

事后小记

我们可以将上述所有操作简单的制作成一个脚本，然后批量执行下，剩下的就是边喝茶边等了。

#!/bin/bash

yum install -y gunzip tar gcc g++ make libtool bison libncurses zlib-dev automake autoconf cmake
cd ~
mkdir packages && cd packages
wget http://mirrors.hustunique.com/mariadb/mariadb-10.0.11/source/mariadb-10.0.11.tar.gz
tar zxf mariadb-10.0.11.tar.gz
mkdir /usr/local/mysql
mkdir /var/run/mysqld
groupadd mysql
useradd -d /usr/local/mysql -s /sbin/false -g mysql mysql
chown -R mysql.mysql /usr/local/mysql
chown -R mysql.root /var/run/mysqld
cd mariadb-10.0.11

这个脚本只是提供一个思路，并不代表一定要这么做，事实上更好的做法，是制作成相应的RPM安装包，然后直接批量推送安装就好了。

关于RPM包的制作方法，请参考https://fedoraproject.org/wiki/How_to_create_an_RPM_package/zh-cn
