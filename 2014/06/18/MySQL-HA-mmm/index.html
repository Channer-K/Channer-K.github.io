MySQL-HA-MMM

MySQL-MMM全称为Multi-Master Replication Manager for MySQL，顾名思义它是基于MySQL双主模型来实现的MySQL的高可用。因此，在安装配置mmm之前需要有一个可以正常工作的双主复制结构数据库群。

服务器规划

为了演示方便，我们采用满足条件的最简单的复制架构，既两主一备的架构。

Function ip hostname server id
monitor 192.168.10.10 monitor
master 1 192.168.10.11 master1 1
master 2 192.168.10.12 master2 2
slave 192.168.10.13 slave 10

从mmm的角度可以将以上4台服务器分为两组：监控端和代理端，即monitor端和agent端。

MySQL双主模型的配置

MySQL双主模型的配置工作不是这篇文章的重点，相关的教程网上也不胜枚举，因此不会在这里用太多的篇幅来讲解，这里只简单的说明下配置过程。

我们接下来的配置过程分为两步：首先，配置完成两台master服务器，接着接入slave服务器。

基本配置

编辑my.cnf文件，添加如下内容到[mysqld]部分：

[mysqld]
...
server_id           = 1
log_bin             = /var/log/mysql/mysql-bin.log 
log_bin_index       = /var/log/mysql/mysql-bin.log.index 
relay_log           = /var/log/mysql/mysql-relay-bin 
relay_log_index     = /var/log/mysql/mysql-relay-bin.index 
expire_logs_days    = 10 
max_binlog_size     = 100M 
log_slave_updates   = 1
auto_increment_increment = 2
auto_increment_offset = 1
...

auto_increment_increment和auto_increment_offset是为了避免自增ID主键重复的。具体说明可以参考官方说明

创建用户

出于安全和管理的要求，我们不可能什么都用root权限的用户来操作，root用户也是在生产环境下尽量避免使用的。

所以我们需要针对不同的功能建立不同权限的用户：

角色 权 限 描述
monitor user REPLICATION CLIENT mmm监控端用来监控MySQL健康状态的账户
agent user SUPER, REPLICATION CLIENT, PROCESS mmm代理端用来改变MySQL状态的账户
replication user REPLICATION SLAVE 同步复制用的账户

在master1上执行下来语句来添加上面那些我们需要的用户：

(master1) mysql> GRANT REPLICATION CLIENT ON *.* TO 'monitor'@'192.168.10.%' \
    IDENTIFIED BY 'monitor_password';

(master1) mysql> GRANT SUPER, REPLICATION CLIENT, PROCESS ON *.* TO\    'agent'@'192.168.10.%' IDENTIFIED BY 'agent_password';

(master1) mysql> GRANT REPLICATION SLAVE ON *.* TO 'replication'@'192.168.10.%' \
    IDENTIFIED BY 'replication_password';

我这里把权限赋予了整个192.168.10段的用户，也就是说整个192.168.10网段的用户都可以通过以上用户接入MySQL，如果你没有办法接收这种比较宽泛的权限限制，也可以更严格和精确到特定主机的IP

同步准备

在开始同步之前，需要先将master1所有的数据库备份，然后导入到其他各个数据库节点中。

(master1) shell> mysqldump -u root -p --all-databases > /root/all.sql

将配置和数据库备份拷贝到其他节点上：

# copy to master2
(master1) shell> scp /etc/my.cnf root@master2:/etc/
(master1) shell> scp /root/all.sql root@master2:/root/

# copy to slave
(master1) shell> scp /etc/my.cnf root@slave:/etc/
(master1) shell> scp /root/all.sql root@slave:/root/

# import data
(master2) shell> mysql -u root -p < /root/all.sql
(slave)   shell> mysql -u root -p < /root/all.sql

注意：配置文件拷贝到各个节点之后，不要忘记修改server_id和auto_increment_offset的值：

# master2
server_id            = 2
auto_increment_offset = 2

# slave
server_id            = 10

启动复制

CHANGE MASTER之前，我们需要知道master当前的bin_log文件名和位置，由于是双主结构，每台master互为对方的主从，因此我们需要知道两台master主机的bin_log文件名和位置：

(master1) mysql> SHOW MASTER STATUS;
+------------------+----------+--------------+------------------+
| File             | Position | Binlog_Do_DB | Binlog_Ignore_DB |
+------------------+----------+--------------+------------------+
| mysql-bin.000002 |      326 |              |                  |
+------------------+----------+--------------+------------------+
1 row in set (0.00 sec)

(master2) mysql> SHOW MASTER STATUS;
+------------------+----------+--------------+------------------+
| File             | Position | Binlog_Do_DB | Binlog_Ignore_DB |
+------------------+----------+--------------+------------------+
| mysql-bin.000002 |      326 |              |                  |
+------------------+----------+--------------+------------------+
1 row in set (0.00 sec)

到这里我们就完成了所有准备工作，下面就可以到各个节点启动复制了。

(master1) mysql> CHANGE MASTER TO master_host='192.168.10.12', 
                                  master_port=3306, 
                                  master_user='replication', 
                                  master_password='replication_password',
                                  master_log_file='<file>', 
                                  master_log_pos=<position>;
(master2) mysql> CHANGE MASTER TO master_host='192.168.10.11', 
                                  master_port=3306, 
                                  master_user='replication', 
                                  master_password='replication_password',
                                  master_log_file='<file>', 
                                  master_log_pos=<position>;
(slave) mysql> CHANGE MASTER TO master_host='192.168.10.11', 
                                master_port=3306, 
                                master_user='replication',
                                master_password='replication_password',
                                master_log_file='<file>', 
                                master_log_pos=<position>;

和请根据SHOW MASTER STATUS的结果进行替换。

分别在master1, master2和slave上启动slave进程：

(master1) mysql> START SLAVE;
(master2) mysql> START SLAVE;
(slave)   mysql> START SLAVE;

检查复制状态是否正常，

(master1) mysql> SHOW SLAVE STATUS\G
*************************** 1. row *************************** 
         Slave_IO_State: Waiting for master to send event 
            Master_Host: 192.168.0.12
            Master_User: replication
            Master_Port: 3306 
          Connect_Retry: 60
            ...
       Slave_IO_Running: Yes
      Slave_SQL_Running: Yes
            ...

(master2) mysql> SHOW SLAVE STATUS\G
*************************** 1. row *************************** 
         Slave_IO_State: Waiting for master to send event 
            Master_Host: 192.168.0.11
            Master_User: replication
            Master_Port: 3306
          Connect_Retry: 60
            ...
       Slave_IO_Running: Yes
      Slave_SQL_Running: Yes
            ...

(slave) mysql> SHOW SLAVE STATUS\G
*************************** 1. row *************************** 
         Slave_IO_State: Waiting for master to send event 
            Master_Host: 192.168.0.11
            Master_User: replication
            Master_Port: 3306 
          Connect_Retry: 60 
            ...
       Slave_IO_Running: Yes
      Slave_SQL_Running: Yes
            ...

只有当Slave_IO_Running和Slave_SQL_Running两个都是Yes的时候才可以继续进行下面的操作，否则还是先解决复制的故障再说高可用吧。

如果你足够幸运的话应该可以得到跟我一样的结果，这说明我们的同步已经正常工作了，那么下面就可以开始安装和配置MySQL-MMM了。

安装MMM

就像安装其他软件一样，安装MySQL-MMM也可以通过麻烦点的编译方式安装，也可以选择简单实用的yum方式安装（需要epel源）。

yum方式可以帮我解决复杂的依赖关系，简化我们的安装过程，我实在想不出来不用它的理由，所以我在这里就选择yum方式来安装了。如果你是ubuntu的操作系统，可以通过MySQL-MMM的官方下载deb包自行安装。你可以点击这里打开下载页面。

我不想在一篇文章里介绍太多不相关的东西，不会配置EPEL源的请各位观众老爷自行百度。

Monitor端也就是监控端，我们要安装monitor组件：

(monitor) shell> yum -y install mysql-mmm-monitor*

Agent端也就是代理端，我们需要安装agent相关的组件：

(master1) shell> yum -y install mysql-mmm-agent*
(master2) shell> yum -y install mysql-mmm-agent*
(slave)   shell> yum -y install mysql-mmm-agent*

配置

Monitor

安装完MySQL-MMM之后会在/etc/mysql-mmm目录下生成一个名为mmm_mon.conf的配置文件，这个配置文件就是monitor的主配置文件，打开并编辑，

include mmm_common.conf

<monitor>
    ip                  127.0.0.1
    pid_path            /var/run/mysql-mmm/mmm_mond.pid
    bin_path            /usr/libexec/mysql-mmm
    status_path         /var/lib/mysql-mmm/mmm_mond.status
    ping_ips            192.168.10.11, 192.168.10.12, 192.168.10.13
    auto_set_online     60

    # The kill_host_bin does not exist by default, though the monitor will
    # throw a warning about it missing.  See the section 5.10 "Kill Host
    # Functionality" in the PDF documentation.
    #
    # kill_host_bin     /usr/libexec/mysql-mmm/monitor/kill_host
    #
</monitor>

<host default>
    monitor_user        monitor
    monitor_password    monitor_password
</host>

debug 0

ping_ips中列出的IP是MMM用来检测monitor的网络连接是否正常的，一般填写所有的MySQL节点的IP（不是虚拟IP）即可。

Agent

安装完MySQL-MMM的agent组件的服务器会在/etc/mysql-mmm目录下生成一个名为mmm_common.conf的配置文件，修改该配置文件：

active_master_role      writer

<host default>
    cluster_interface       eth0
    pid_path                /var/run/mysql-mmm/mmm_agentd.pid
    bin_path                /usr/libexec/mysql-mmm/
    replication_user        replication            # replication_user和agent_user
    replication_password    replication_password  # 就是上面创建用户那一小节中创建的用户
    agent_user              agent                 
    agent_password          agent_password       
</host>

<host db1>
    ip      192.168.10.11
    mode    master
    peer    db2
</host>

<host db2>
    ip      192.168.10.12
    mode    master
    peer    db1
</host>

<host db3>
    ip      192.168.10.13
    mode    slave
</host>

<role writer>
    hosts   db1, db2
    ips     192.168.10.100
    mode    exclusive
</role>

<role reader>
    hosts   db2, db3
    ips     192.168.10.101, 192.168.10.102
    mode    balanced
</role>

\和\部分的ips指的是虚拟IP。之所以要配置虚拟IP，是因为转移虚拟IP要比转移真实的网卡IP容易得多，MySQL-MMM通过检测数据库的状态来发现故障主机，并将虚拟IP从故障主机转移到健康主机，从而实现高可用的，虽然这种说法不是很严谨，但用来帮助理解还是足够的。

将配置完成的mmm_common.conf文件复制到其他两个MySQL节点。

(master1) shell> scp /etc/mysql-mmm/mmm_common.conf root@192.168.10.12:/etc/mysql-mmm/
(master1) shell> scp /etc/mysql-mmm/mmm_common.conf root@192.168.10.13:/etc/mysql-mmm/

在MySQL节点还有一个配置文件需要修改，那就是/etc/mysql-mmm目录下的mmm_agent.conf文件。

include mmm_common.conf
this db1

还记得在mmm_common.conf文件中我们定义3个host段吧，这个host端的配置就是为了告诉monitor端它要监控哪些机器用的，而mmm_agent.conf的this参数就是告诉monitor端当前节点对应了mmm_common.conf中哪个host。例如，host db1对应了master1这台机器，因此，mater1这台机器的mmm_agent.conf的this就是为db1。同理，

# master2
include mmm_common.conf
this db2

# slave
include mmm_common.conf
this db3

启动

完成上述配置之后，就可以尝试启动MMM了。

# 启动agent端
(master1) shell> /etc/init.d/mysql-mmm-agent start
(master2) shell> /etc/init.d/mysql-mmm-agent start
(salve)   shell> /etc/init.d/mysql-mmm-agent start

# 启动monitor端
(monitor) shell> /etc/init.d/mysql-mmm-monitor start

我们可以通过mmm_control命令来查看集群的状态。

(monitor) shell> mmm_control show
  db1(192.168.10.11) master/ONLINE. Roles: writer(192.168.10.100)
  db2(192.168.10.12) master/ONLINE. Roles: reader(192.168.10.101)
  db3(192.168.10.13) slave/ONLINE. Roles: reader(192.168.10.102)
